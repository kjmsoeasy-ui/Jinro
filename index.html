Section 01

프로젝트 개요

이 웹페이지는 텍스트 생성 LLM, 이미지 생성 엔진 및 TTS와 상호 작용할 수 있는 로컬 설치형 사용자 인터페이스인 SillyTavern에 적용하기 위해 직접 제작한 커스텀 CSS 테마 코드를 분석한 결과물이다. SillyTavern은 자체 테마 시스템을 제공하며, 사용자가 CSS를 직접 작성하여 인터페이스 전체를 재스타일링할 수 있다.

해당 테마는 파스텔 핑크와 하늘색의 그라데이션 조합을 중심으로, 한국어 손글씨 계열 폰트를 적용하여 부드럽고 아기자기한 느낌의 채팅 UI를 구현하는 것을 목표로 설계되었다.

제작 목적 : 기존 디폴트 테마의 어두운 톤을 탈피하고, 사용자 경험 측면에서 시각적으로 따뜻하고 즐거운 감성의 인터페이스를 직접 구현해보는 것. UI/UX 디자인의 실질적 구현 능력을 탐색하는 진로 과제의 일환이기도 하다.

Section 02

디자인 컨셉 및 색채 계획

테마 전반의 비주얼 방향성은 "파스텔 색상의 무지개"로 요약된다. 두 개의 핵심 색조인 Pastel Pink와 Pastel Blue가 교차·혼합되며 인터페이스 전체에 부드러운 그라데이션을 형성한다.

색상 정의는 CSS :root 의 CSS Custom Properties(변수)로 중앙화되었다. 이 방식은 색상 변경 시 한 곳만 수정하면 전체에 반영되는 유지보수 효율을 제공한다. UI/UX 설계에서 디자인 토큰(Design Token) 개념의 실질적 적용이다.

색상 변수 정의 — :root
:root {
  --pastel-pink: #FFD1DC;
  --pastel-blue: #B3E5FC;
  --soft-pink:   #FFB7CE;
  --soft-blue:   #90CAF9;
}

사용자(유저) 말풍선과 캐릭터 말풍선에 각각 다른 색상을 배정한 것도 설계 포인트다. 유저는 핑크, 캐릭터는 하늘색으로 역할을 시각적으로 구분하여 메시지 출처의 직관적 인식을 돕는다. 이는 채팅 UI의 대표적인 UX 관례를 감성적으로 재해석한 것이다.

Section 03

핵심 기술 구현 분석

CSS 기술 요소 코드 내 적용 의도 및 효과
@font-face 학교안심 나들이체 woff2 로드 외부 CDN에서 한국어 웹폰트를 직접 불러와 브라우저 기본 폰트를 대체. 감성적 분위기 형성의 핵심.
CSS 변수 --pastel-pink, --soft-blue 등 색상 토큰 중앙화. 팔레트 일관성 유지 및 유지보수 효율 향상.
속성 선택자 [is_user="true"], :not([is_user="true"]) HTML 커스텀 속성을 기반으로 유저/캐릭터 말풍선을 다르게 스타일링. 역할 구분의 핵심 선택자.
linear-gradient 전송창, 사이드바, 슬라이더 트랙 등 핑크→블루 수평 그라데이션으로 "파스텔 무지개" 테마 전체에 통일감 부여.
@keyframes heartBeat 애니메이션 정지 버튼(#mes_stop)에 1초 주기 심장 박동 효과. 인터랙션 피드백과 감성적 연출의 결합.
가상 선택자 ::before, ::after, content 정지 버튼 아이콘을 ♥로 교체. HTML을 수정하지 않고 CSS만으로 아이콘 변경.
스크롤바 커스텀 ::-webkit-scrollbar-thumb 스크롤바에도 핑크→블루 그라데이션 적용. 인터페이스 전체 감성 통일.
슬라이더 커스텀 input[type="range"], ::-webkit-slider-thumb 기본 브라우저 슬라이더를 완전히 재디자인. 트랙은 그라데이션, 손잡이는 핑크 테두리 원형.
box-shadow 말풍선 .mes_text 미세한 그림자로 말풍선에 깊이감 부여. 배경과의 자연스러운 분리.
transition 버튼 hover 시 0.3s 전환으로 hover 상태의 색 변화를 부드럽게 처리. 인터랙션 품질 향상.

특히 주목할 부분은 content 속성을 통한 아이콘 교체다. HTML 구조를 건드리지 않고 CSS의 ::before 가상 요소와 content: "\f004" (Font Awesome 하트 유니코드)만으로 아이콘을 변경한 것은, CSS가 단순한 스타일링을 넘어 UI 요소를 재정의하는 수단이 될 수 있음을 보여주는 사례다.

아이콘 교체 — content 속성 활용
/* 정지 버튼 아이콘을 하트로 교체 */
#mes_stop i::before {
  content: "\f004" !important;     /* Font Awesome 하트 코드 */
  font-family: "Font Awesome 6 Free" !important;
  font-weight: 900 !important;
}

/* 두근두근 애니메이션 */
#mes_stop {
  animation: heartBeat 1s ease-in-out infinite !important;
}

@keyframes heartBeat {
  0%, 100% { transform: scale(1); }
  50%       { transform: scale(1.2); }
}
Section 04

UI/UX 설계 관점 분석

이 테마를 UI/UX 설계 원칙의 관점에서 분석하면 다음과 같은 특징이 나타난다.

🎭
역할의 시각적 구분

유저 핑크 / 캐릭터 하늘 — 색상을 통한 발화 주체 즉각 인식. 채팅 UX의 핵심 원칙 준수.

🎞
감성적 일관성

그라데이션, 점선 테두리, 폰트, 스크롤바까지 테마 감성이 인터페이스 전체에 통일되어 있음.

✨
마이크로 인터랙션

heartBeat 애니메이션, hover 트랜지션 — 기능적 요소에 감정을 부여하는 UX 세부 설계.

🔧
!important 과다 사용

SillyTavern의 높은 명시도를 덮기 위한 불가피한 선택이나, 코드 유지보수 측면의 단점이기도 함.

UX 관점 핵심 : 이 테마는 단순히 "예쁘게 꾸미기"가 아닌, 사용자가 AI와 대화하는 경험의 질을 설계한 것이다. 색상·폰트·애니메이션의 조합이 사용자에게 심리적 안락함과 몰입감을 제공하도록 의도되었다.

전송 폼(#send_form)의 레이아웃 처리도 주목할 만하다. display: flex와 align-items: center를 조합하여 입력창, 아이콘 버튼들을 수직 정렬하고, 외부 그라데이션 컨테이너 안에 흰색 입력창을 배치하는 구조를 설계하였다.

전송 폼 Flexbox 레이아웃
#send_form {
  display: flex !important;
  align-items: center !important;        /* 수직 중앙 정렬 */
  justify-content: space-between !important;
  background: linear-gradient(90deg, #f6d4e3, #d1ebf6) !important;
  border-radius: 30px !important;
}

#send_textarea {
  flex: 1 !important;                    /* 남은 공간 차지 */
  background-color: white !important;
  border-radius: 15px !important;
}
Section 05

개선 가능성 및 발전 방향

완성도 높은 첫 번째 결과물이지만, 코드 품질과 UX 측면에서 발전시킬 수 있는 방향이 있다.

1
중복 선언 정리

#send_form, input[type="range"] 등 동일 요소가 여러 곳에 반복 선언되어 있다. CSS를 구조화하거나 주석으로 구역을 명확히 나누면 유지보수성이 크게 향상된다.

2
다크 모드 대응

@media (prefers-color-scheme: dark)를 활용하여 시스템 다크 모드 시에도 감성이 유지되는 어두운 파스텔 버전을 추가하면 UX의 폭이 넓어진다.

3
반응형 고려

모바일/태블릿 환경에서의 레이아웃 대응을 위해 @media 쿼리를 추가하면 더 많은 환경에서 테마가 올바르게 표현된다.

4
접근성(Accessibility) 검토

파스텔 계열은 명도 대비가 낮아 시각적 접근성이 부족할 수 있다. WCAG 기준 대비율 확인 후 텍스트 색상을 보정하면 더 완성도 높은 UI가 된다.

5
CSS 변수 확장

현재 색상 변수만 정의되어 있다. --radius-sm, --shadow-default 등 간격·그림자 값도 변수화하면 테마 전체를 더 일관성 있게 관리할 수 있다.

Section 06

결론

이 테마가 보여주는 것

이번 SillyTavern 커스텀 테마 제작은 단순한 꾸미기 활동이 아니다. CSS 변수를 통한 디자인 시스템 설계, 속성 선택자를 통한 역할 기반 스타일링, 애니메이션을 통한 마이크로 인터랙션 구현 — 이 모든 요소가 UI/UX 설계의 실질적 핵심 개념들과 맞닿아 있다.

사용자가 인터페이스를 어떻게 느끼는가를 고민하고, 그것을 코드로 구현한 경험은 디지털미디어학과에서 탐구할 인간 중심 설계(Human-Centered Design)의 출발점이 된다. 이 보고서가 그 탐구의 첫 문서가 되기를 바란다.
